# Water Bottle Production Line Simulation

This repository contains the final version of the Water Bottle Production Line Simulation project for the Advanced Programming course.

The project simulates a water bottle manufacturing process with a Python backend, Tkinter HMI, InfluxDB telemetry storage, Grafana dashboard visualization, and Docker-based infrastructure.

## Project Overview

The simulation represents a continuous water bottle production line. Bottles move through the following stages:

1. Bottle Creation
2. Bottle Quality Control
3. Water Filling
4. Fill Quality Control
5. Cap Installation
6. Cap Quality Control
7. Label Application
8. Label Quality Control
9. Final Quality Control
10. Packaging
11. Shipping

The system includes quality control, rework handling, final rejection, production counters, KPI calculations, sensor simulation, CSV export, and live dashboard monitoring.

## Main Files

| File | Description |
|---|---|
| `production_lineV7.py` | Backend logic for the production line simulation. Includes bottle creation, production stages, defect detection, rework, rejection, KPI calculations, simulated sensors, telemetry snapshot generation, and CSV export. |
| `hmi_v7.py` | Tkinter HMI used to operate and monitor the production line. Includes Start, Stop, Reset, Export CSV, animated stage highlighting, live counters, quality log, and production history. |
| `producer.py` | Reads the latest telemetry from `telemetry_state.json` and writes production, sensor, and machine-state data to InfluxDB once per second. |
| `docker-compose.yml` | Starts InfluxDB and Grafana containers. |
| `requirements.txt` | Python dependencies needed by the project. |
| `production_report_v7.csv` | Example/exported production report generated by the HMI. |

- Python 3.10 or newer
- Docker Desktop
- Git

Install Python dependencies:

```bash
pip install -r requirements.txt
```

## How to Run the Project

### 1. Start InfluxDB and Grafana

Open a terminal in the project folder and run:

```bash
docker compose up -d
```

InfluxDB will run at:

```text
http://localhost:8086
```

Grafana will run at:

```text
http://localhost:3000
```

Default Grafana login:

```text
Username: admin
Password: admin
```

### 2. Run the HMI

In a terminal, run:

```bash
python hmi_v7.py
```

Use the HMI buttons to start, stop, reset, and export production data.

### 3. Run the Telemetry Producer

Open a second terminal in the same project folder and run:

```bash
python producer.py
```

The producer reads the HMI telemetry and sends the data to InfluxDB.

### 4. Open Grafana

Open Grafana in the browser:

```text
http://localhost:3000
```

The dashboard displays production counters, quality KPIs, runtime, production rate, water temperature, tank fill level, and machine state.

## InfluxDB Settings

The project uses the following InfluxDB settings:

| Setting | Value |
|---|---|
| URL | `http://localhost:8086` |
| Organization | `lecture` |
| Bucket | `factory` |
| Token | `my-super-secret-token` |

When connecting Grafana to InfluxDB inside Docker, use:

```text
http://influxdb:8086
```

## Dashboard Measurements

The producer writes three main measurements:

| Measurement | Data |
|---|---|
| `water_bottle_production` | Production counters and KPIs |
| `water_bottle_sensors` | Simulated sensor values |
| `water_bottle_state` | Machine state and current stage |

## CSV Export

The HMI can export production history to:

```text
production_report_v7.csv
```

The CSV report includes:

- Time
- Bottle ID
- Result
- Stage
- Reason

## Notes

This is a simulation project. It does not connect to real sensors or PLC hardware. Sensor values such as water temperature, conveyor speed, motor current, and tank level are simulated in Python.
